Question compilation pipeline

Question handling begins with normalization. The input is turned into an event span with explicit markers for interrogative intent, making the target of compilation explicit before any search begins.

Retrieval uses VSA to propose schemas and macro programs that resemble the question. These candidates define program structure and typed slots, which are filled through discrete matching, coreference, and associative hints.

Multiple compiled programs are kept in a beam and tested for explanatory fit and early consistency. This ensures interpretive uncertainty is managed explicitly rather than ignored.

VSAVM is an Axiologic Research experiment within the Achilles project. This static documentation is written in clear academic English for engineers.
