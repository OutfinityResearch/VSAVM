Executable core architecture

The VM is the core of the system and must remain simple, explicit, and typed. Its state includes a canonical fact store, a rule library, a binding environment, and an execution log for audit.

The instruction set is intentionally compact, focusing on construction, unification, branching, and conflict checks. This keeps execution understandable and easier to test.

VSA functions as a retrieval accelerator. It proposes candidates and shortlists, while the VM remains the arbiter of truth through execution and bounded closure.

VSAVM is an Axiologic Research experiment within the Achilles project. This static documentation is written in clear academic English for engineers.
