Executable core architecture

The VM is the core of the system and must remain simple, explicit, and typed. It maintains a canonical fact store, a rule library, a binding environment, and an execution log that can be inspected for audit.

The instruction set is intentionally compact, focusing on construction, unification, branching, and conflict checks. This keeps execution understandable and makes the system easier to reason about and test.

VSA functions as a retrieval accelerator. It proposes candidates and shortlists, while the VM remains the arbiter of truth through execution and bounded closure.

VSAVM is an Axiologic Research experiment within the Achilles project. This static documentation is written in clear academic English for engineers.
