Consistency contract and boundary behavior

The consistency contract defines how the system behaves at the edge of its computational budget. It formalizes what it means for a conclusion to be robust, conditional, or indeterminate.

Closure runs in two phases, propagation and verification, which separates derivation from conflict detection. Strict mode prunes inconsistent branches, while exploratory mode can keep them for analysis without letting them influence the final answer.

When resources are exhausted, the system must degrade gracefully. It does not hallucinate; it either returns a conditional statement or explicitly says that the budget was insufficient for a definitive answer.

VSAVM is an Axiologic Research experiment within the Achilles project. This static documentation is written in clear academic English for engineers.
